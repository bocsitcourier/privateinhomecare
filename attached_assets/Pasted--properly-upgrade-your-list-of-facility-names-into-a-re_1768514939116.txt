Phase 1: The "Data Enrichment" Prompt
This prompt instructs the Replit Agent to build a script that takes your list of names, finds them on Google Maps, and downloads all the missing metadata into a structured JSON file.

Copy/Paste this into Replit Agent:

Role: Senior Data Engineer & API Specialist.

Objective: Create a Node.js script to "enrich" a list of Massachusetts nursing facility names with real-time data from the Google Places API (New Version).

Input Data:

Read from facilities.txt (A line-separated list of facility names).

API Integration:

Use the Google Places API (New) endpoint: https://places.googleapis.com/v1/places:searchText

Authentication: Use dotenv for GOOGLE_PLACES_API_KEY.

Field Masking (Crucial for Cost/Speed): Only request these specific fields:

places.formattedAddress

places.nationalPhoneNumber

places.rating

places.userRatingCount

places.googleMapsUri (The direct link to the map)

places.reviews (Get the top 3 most relevant reviews)

places.websiteUri

Logic & Flow:

Concurrency: Use p-limit (set to 10) to process the list quickly but within API rate limits.

Search Logic: Send the facility name + "Nursing Home Massachusetts" as the text query to ensure accuracy.

Validation: If the API returns no results, log it to not_found.txt so I can fix the name manually.

Output: Save the successful results to enriched_facilities.json.

Structure: The JSON should be an array of objects where each object combines the original name with the new API data.

Sample Output Structure:

JSON

[
  {
    "originalName": "Example Manor",
    "address": "123 Main St, Boston, MA",
    "phone": "(617) 555-0199",
    "mapsLink": "https://maps.google.com/...",
    "rating": 4.5,
    "reviews": [ ... ],
    "reviewCount": 120
  }
]
Execution: Install axios, dotenv, and p-limit. Scaffold the script.
